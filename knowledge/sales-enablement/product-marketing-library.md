# Product Marketing Library

Use this library with `knowledge/sales-enablement/output-pack.md` when generating product-specific sales assets.

## Shared Messaging Architecture

Aevah and Pruvida help organizations turn fragmented operational knowledge, data, processes, and systems into trusted, intelligent, AI-enabled business capabilities.

## Offer Roles

| Offer Layer | Role |
| --- | --- |
| Aevah | Platform and product layer for data products, AI-ready semantics, MDM, monitoring, and migration. |
| Pruvida | Consulting, strategy, engineering, implementation, and advisory layer. |
| Blueprint Studio | AI-assisted product and delivery workspace for turning intent into buildable work. |

## Standard Marketing Sections

Each product page should include:

- Marketing source references
- Messaging pillars
- Use cases
- Packaging ideas
- Sales motion
- Recommended metrics
- Objections and proof needed

## Suggested Asset Types

- Executive one-pager
- Buyer-specific executive narrative
- Discovery call guide
- Qualification checklist
- Proposal draft
- SOW draft
- Objection battlecard
- ROI hypothesis
- Competitive positioning brief
- Demo script
- Email follow-up sequence
- Website landing page draft
- Partner enablement brief
- Federal capability statement
- Board-ready transformation memo

## Agent Use

When generating a sales asset:

1. Load the product page.
2. Use the product's marketing pillars and package ideas.
3. Select metrics that match the prospect's pain and stage.
4. Avoid claims that exceed the product page.
5. If a page has no proof points, mark proof as needed rather than inventing it.
